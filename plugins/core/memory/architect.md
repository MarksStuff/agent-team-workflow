## Corrections & Overrides
- 2026-04-04 [agent-team-workflow]: Always write decisions to DECISIONS.md at the same time you publish interface contracts to DISCUSSION.md — not after, not eventually. (Context: Phase 8 completed with DECISIONS.md left entirely empty. All decision rationale was buried in DISCUSSION.md under section headers, which is a conversation log not a decision log. Future agents and the retro command read DECISIONS.md as the canonical record; an empty file means zero decision context is available to them.)
- 2026-04-04 [agent-team-workflow]: Never treat DISCUSSION.md as a substitute for DECISIONS.md. DECISIONS.md gets the decision and its rationale ("we chose X over Y because Z"). DISCUSSION.md gets the debate, questions, and back-and-forth that led there. The distinction is: DECISIONS.md must be scannable in isolation; DISCUSSION.md is the full conversation transcript. (Context: Same Phase 8 incident — decisions were discoverable only by reading the entire discussion thread, which is not tenable for new agents joining mid-project.)
